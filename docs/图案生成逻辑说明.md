# 图案生成逻辑说明

## 整体流程

1. **用户选色** → 调色板 = 用户所选颜色 + 自动添加的黑、白
2. **随机抽 3 个主题** → 从 10 个提示词模板里随机选 3 个（如：可爱动物、表情包、植物…）
3. **调用 AI 画图** → 每个主题用「模板描述 + 颜色列表 + 统一风格/背景/描边说明」生成一张图
4. **像素化** → 横轴 50 格，纵轴按比例；按「主色」把每个格子映射到调色板里最接近的颜色
5. **背景留空** → 把与四边连通的背景色标成「不填色」
6. **去孤立豆** → 只保留最大一块连通区域，其余标成空（边缘密封、无散豆）
7. **统计与输出** → 算各色数量、总豆数，用于预览和下载

## 导致质量不稳、画风不统一的原因

- **每次随机 3 个主题**：主题差异大，画风容易不一致
- **“只用这些颜色”**：强制用少量 hex，AI 容易配出不好看的图
- **提示词过长、规则多**：颜色 + 风格 + 描边混在一起，模型容易顾此失彼
- **没有统一画风约束**：各模板只写主题，没有统一的「画风」描述

## 已做的改进（代码内）

- **统一画风前缀**：所有生成都先加同一段「扁平、可爱、适合拼豆」的风格描述
- **简化颜色要求**：改为「优先使用这些颜色」而不是「只能用」，减少翻车
- **精简提示词**：主题 + 风格 + 背景，描边单独一句，避免冗长
