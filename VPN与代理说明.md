# 为什么开了 VPN 还是失败？

## 原因：谁在访问 Google？

- 你在浏览器里点的「生成」→ 请求发到你**本机的 Next 服务器**（localhost）
- 真正去访问 **Google API** 的是你电脑上的 **Node.js 进程**，不是浏览器

所以：
- 浏览器可能已经通过 VPN 上网
- 但 **Node 进程** 可能没有走 VPN，还是直连，所以会「网络连接失败」

## 两种常见情况

### 1. VPN 是「浏览器插件」或只代理部分软件
- 只有浏览器流量走 VPN
- Node 不走 VPN → 连不上 Google → 一直失败

### 2. VPN 是「系统级 / 全局」
- 理论上 Node 也会走 VPN
- 若仍失败，可能是：VPN 没真正全局、或需要再配「本地代理」给 Node 用（见下文）

## 解决办法：让 Node 走代理

很多 VPN 或翻墙软件会提供一个 **本地 HTTP 代理**（本机地址 + 端口），让其他程序通过它上网。  
只要让 **Next 项目里的 Node** 使用这个代理，就能访问 Google。

### 步骤 1：找到本地代理地址

常见软件的默认端口（在设置里可查）：

| 软件 | 常见代理地址 |
|------|--------------|
| Clash / Clash for Windows | `http://127.0.0.1:7890` |
| V2RayN | `http://127.0.0.1:10808` |
| Shadowsocks | `socks5://127.0.0.1:1080` |
| 其他 | 在软件设置里找「HTTP 代理端口」或「本地监听」 |

### 步骤 2：在项目里配置代理

编辑项目根目录的 **`.env.local`**，**取消注释**下面两行，并改成你的代理地址（端口按你实际为准）：

```env
HTTPS_PROXY=http://127.0.0.1:7890
HTTP_PROXY=http://127.0.0.1:7890
```

例如用 Clash 就改成 7890，用 V2RayN 就改成 10808。

### 步骤 3：重启 Next 开发服务器

- 关掉当前运行的 `npm run dev`（Ctrl+C）
- 再执行一次：`npm run dev`

然后再在页面上点「生成」试一次。

### 步骤 4：确认是否生效

重启后，看终端日志里是否出现类似：

- `✅ 代理配置成功: http://127.0.0.1:7890`
- `使用代理连接Google Gemini API`

有这两句说明 Node 已经在走代理，再失败就多半是代理本身或 VPN 没连好的问题。

## 小结

- 失败是因为 **连 Google 的是 Node，不是浏览器**，Node 可能没走你的 VPN。
- 解决办法：在 `.env.local` 里配置 `HTTPS_PROXY` / `HTTP_PROXY` 指向你 VPN/翻墙软件的**本地代理**，然后**重启服务器**。
